## Purpose

This repository is designed to help students prepare for hands-on learning experiences involving Python, Jupyter notebooks, data, classic AI / machine learning, and agentic AI.

The goal is not just to get answers. The goal is to help students learn how to:
- understand the problem
- use AI and coding tools wisely
- test and verify results
- improve weak solutions
- explain what they are doing
- work through scaffolded notebook challenges with confidence

## Who This Is For

This repository is designed for:
- high school students
- interns
- beginner programmers
- mentors and coaches supporting student teams
- anyone preparing for hands-on AI learning activities

## Learning Philosophy

Students should learn enough to think, question, test, and solve well with AI.

AI can help explain, suggest, and generate. Students still need to:
- understand the task
- recognize when something is wrong
- adapt code to the situation
- verify outputs
- explain why a solution works

A strong learner is not the one who copies the fastest. A strong learner is the one who can combine human judgment with AI assistance to solve problems responsibly.

## What Students Should Be Ready To Do

By the time students begin challenge-style notebooks, they should be able to:
- run Jupyter notebook cells in order
- read starter code and complete TODO sections
- use Python functions, lists, dictionaries, and loops
- inspect tables and basic data
- debug small errors without panicking
- ask AI for help in a focused way
- test outputs instead of trusting them blindly
- explain what they changed and why

## Learning Paths

### Shared Foundation
All students should begin with:
- Jupyter notebook basics
- Python basics
- reading and debugging code
- data basics
- step-by-step problem solving
- reading scaffolded notebook tasks
- basic plotting and interpretation
- using AI well while learning

### Classic AI Path
This path focuses more on:
- data exploration
- tabular data
- preprocessing
- feature understanding
- model workflows
- anomaly detection and evaluation
- edge cases and guardrails
- interpreting results

### Agentic AI Path
This path focuses more on:
- prompts
- large language models
- tools and functions
- context handling
- workflows
- multi-step reasoning
- validation and guardrails
- evaluating weak or misleading outputs

## How To Use This Repo

A good approach is:
1. Read the summary
2. Complete the foundations guide
3. Choose one focused path
4. Use the support guides to strengthen weak areas
5. Practice in Jupyter notebooks
6. Use AI as support, not autopilot
7. Test, verify, and improve your work
