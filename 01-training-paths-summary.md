## How the Repository Works

Students begin with shared foundations, then branch into either the Classic AI path or the Agentic AI path.

The support guides are there to strengthen practical skills that show up often in challenge-style Jupyter notebooks:
- reading and editing starter code
- working with pandas and tabular data
- calling APIs and handling JSON
- testing, debugging, and verifying outputs
- reading and interpreting simple charts

This is not a memorize-and-repeat pathway.

The overall goal is to help students:
- learn enough to understand the problem
- use AI as a helper, not autopilot
- verify results carefully
- recover from mistakes
- solve practical tasks with confidence

## Path 1: Shared Foundation

All students should begin with the Foundations Guide.

Core skills include:
- working in Jupyter notebooks
- variables, lists, dictionaries, loops, and functions
- reading and debugging code
- understanding rows, columns, and simple tables
- breaking tasks into smaller steps
- using AI carefully while still thinking for themselves
- reading scaffolded TODO-style notebook problems
- basic plotting and output interpretation

## Path 2: Classic AI / Machine Learning

This path prepares students for work that involves data, analysis, simple model workflows, and evaluation.

Students should build skill in:
- loading and exploring datasets
- using pandas to inspect and clean data
- choosing useful features
- understanding train, validation, and test ideas
- scaling and transforming data
- spotting anomalies or unusual patterns
- evaluating outputs
- testing edge cases
- adding simple rule-based guardrails
- summarizing results clearly

Helpful support guide:
- [07 pandas and Data Basics](./07-pandas-and-data-basics.md)

## Path 3: Agentic AI

This path prepares students for work that involves prompts, tools, workflows, and multi-step AI systems.

Students should build skill in:
- understanding how LLM-powered systems work
- writing and improving prompts
- using Python functions as tools
- passing context from one step to the next
- structuring workflows
- checking output format and quality
- refining prompts after weak results
- testing for reliability and safety
- debugging agent behavior

Helpful support guides:
- [06 Python for Jupyter Notebook Challenges](./06-python-for-jupyter-notebook-challenges.md)
- [08 APIs, JSON, and Live Data](./08-apis-json-and-live-data.md)

## Recommended Study Order

For most students, a good sequence is:

1. Read the summary
2. Complete the foundations guide
3. Read the “Using AI Well” guide
4. Strengthen Python notebook skills
5. Strengthen data skills
6. Move into Classic AI or Agentic AI
7. Practice challenge-style notebook work

## What Students Commonly Need More Practice In

Based on typical challenge work, the most common weak areas are:
- reading starter code without breaking it
- pandas basics
- API and JSON handling
- debugging small errors
- testing one step at a time
- interpreting results instead of just producing them
- explaining what changed and why

## Final Thought

A good student does not need to memorize everything.

A good student needs to be able to:
- understand the task
- use the tools
- think critically
- test carefully
- improve the solution
- use AI wisely without depending on it blindly
