# 16 Using Free AI Tools

This guide helps students use free AI tools effectively while working through Data and Agents Lab.

You do **not** need a paid AI account to practice well.

The goal is not to use the most powerful tool.
The goal is to ask good questions, test carefully, and keep learning.

---

## Free Is Fine

A free AI account is enough for this repository.

You can still use AI to:
- explain code
- help debug errors
- compare two approaches
- improve a prompt
- help you think through the next step

Paid accounts are **not required** for the practice notebooks in this repo.

---

## What Free AI Tools Are Good For

Free AI tools work best when you ask:
- one clear question at a time
- about one bug, one concept, or one step
- for explanation, debugging help, or comparison
- for help improving something you already tried

Good examples:
- What does this error message mean?
- What is this code doing step by step?
- Why is my filter returning zero rows?
- Can you help me improve this prompt?
- What should I test next?
- Can you explain this in simpler language?

---

## What Free AI Tools Are Not Good For

Free AI tools are **not** the best choice for:
- pasting entire notebooks and asking for all the answers
- depending on long conversations to remember everything
- trusting the first answer without testing it
- submitting code you do not understand
- replacing your own thinking

Even a strong AI answer can still be wrong, incomplete, or too generic.

---

## Ask Small, Focused Questions

A focused question is better than a giant paste.

### Weak question
> Solve this notebook for me.

### Better question
> I wrote a function to flag high temperature rows, but it still returns `normal` for 90. What should I check?

### Better question
> I filtered my DataFrame for `status == "review"` and got zero rows. What are three likely causes?

### Better question
> Here is my prompt. How can I make the output format more specific?

Smaller questions help you:
- get better answers
- use fewer messages
- stay in control of the task
- learn faster

---

## Keep Your Own Notes

Free AI tools may have limits.

That means:
- do not depend on the AI to remember everything
- save useful ideas in your own notes
- copy important code into your notebook
- write down what worked and what did not

A simple notebook or document with:
- useful prompts
- common errors
- fixes
- good examples

can help a lot.

---

## Do Not Paste Sensitive Information

If you are using a public free AI tool:
- do not paste private information
- do not paste personal student data
- do not paste restricted project information
- do not paste anything your teacher, school, mentor, or organization says should stay private

When in doubt, leave it out.

You can usually ask a safe version of the same question by:
- removing names
- removing IDs
- shortening the example
- replacing real data with a small sample

---

## Always Verify the Answer

This is one of the most important rules in the whole repository.

**AI helps generate. Humans must decide.**

That means you still need to:
- run the code
- inspect the output
- compare expected vs actual
- test edge cases
- explain what changed
- decide whether the answer makes sense

A good answer is not just something that sounds smart.
A good answer is something you tested.

---

## Good Uses of AI

Good use looks like:
- asking what an error means
- asking for a simpler explanation
- asking how to inspect data
- asking what to test next
- asking how to improve a weak prompt
- asking for help comparing two options

These uses still keep **you** in charge.

---

## Weak Uses of AI

Weak use looks like:
- copy/pasting full answers without reading them
- changing lots of code without understanding it
- asking AI to do all the thinking
- trusting confident output without checking it
- submitting work you cannot explain

That may look fast at first, but it usually hurts learning.

---

## A Good Student Workflow

When you use AI, try this pattern:

1. Read the task
2. Try the problem yourself first
3. Find the specific part you do not understand
4. Ask one focused AI question
5. Test the answer
6. Inspect the result
7. Explain what changed and why

This is much stronger than asking AI to do everything.

---

## Sample Questions Students Can Reuse

### Debugging
- What does this error message mean?
- What should I inspect first?
- Why is this variable missing?
- Why is this function returning the wrong value?

### Data Work
- How do I check for missing values in this DataFrame?
- Why is this filter returning no rows?
- Which columns seem most useful for this task?
- How do I sort by highest temperature?

### Agentic AI
- How can I make this prompt more specific?
- What is weak about this output?
- What should a tool function return here?
- What information needs to be passed to the next step?

### Reflection
- How can I explain this result in plain language?
- What edge case should I test?
- What are two likely reasons this answer could still be wrong?

---

## Final Reminder

You do not need a paid account to learn well.

You do need to:
- ask better questions
- think carefully
- test your results
- keep your own notes
- use AI as support, not autopilot

That is how free AI tools become genuinely useful for learning.
